**Версия:** 2.0 (актуализировано для v2.6.0)
**Дата:** 9 апреля 2026
**Оригинал:** v1.0 от 7 апреля 2026

---

## 📋 Общая цель

Упрощение архитектуры проекта для:
- Снижения потребления ресурсов на роутере (128MB RAM)
- Упрощения поддержки и отладки
- Устранения дублирования кода
- Уменьшения количества зависимостей

## 🔴 ПРИОРИТЕТ 1: Удаление Hysteria2

**Статус:** Всё ещё в коде (parsers + config generation)
**Почему сейчас:** Hysteria2 — специализированный протокол для мобильных сетей, на Wi-Fi/Ethernet не даёт преимуществ перед VLESS

**Что удалить:**
- [ ] `parse_hysteria2_key()` и `hysteria2_config()` из `core/services.py` (~100 строк)
- [ ] Упоминания Hysteria2 из `keys.html` (плитка сервиса)
- [ ] `routes_vpn.py` — обработка Hysteria2 в `get_services_status()`
- [ ] `constants.py` — константы HYSTERIA2_*
- [ ] Если есть init скрипт `S22hysteria2` на роутере
- [ ] ipset `unblockhysteria2` из всех упоминаний

**Экономия:** ~10MB RAM, ~150 строк кода, упрощение UI

## 🟡 ПРИОРИТЕТ 3: Объединение Shadowsocks + Trojan в "Прокси"

**Статус:** Два отдельных сервиса с дублирующейся логикой
**Почему сейчас:** Оба делают SOCKS5 прокси на локальном порту, разница только в протоколе

**Как:**
```json
{
  "protocol": "ss",
  "server": "example.com",
  "port": 443,
  "password": "...",
  "method": "chacha20-ietf-poly1305"
}
```

**Что изменить:**
- [ ] Создать единый парсер `parse_proxy_key()` (SS/Trojan)
- [ ] Создать единый генератор `proxy_config()` с полем `protocol`
- [ ] Заменить `unblocksh` + `unblocktroj` на `unblockproxy` ipset
- [ ] Обновить `keys.html` — одна плитка "Прокси" с выбором протокола
- [ ] Объединить маршруты в `routes_vpn.py`
- [ ] Создать единый init скрипт `S22proxy` (или обновить существующий)
- [ ] Обновить `constants.py` — заменить SS/TROJAN на PROXY

**Экономия:** ~200 строк дублирующегося кода, упрощение UI

- [ ] `unblock_dnsmasq.sh` → вызов `DnsmasqManager.generate_*()`
- [ ] `unblock_update.sh` → вызов `DnsmasqManager` + `IptablesManager`
- [ ] `refresh_ipset.sh` → вызов `refresh_ipset_from_file()` из `services.py`

### 4.2 Создать:
- [ ] `core/unblock_manager.py` — единый интерфейс для всех unblock операций
- [ ] Обновить `S99unblock` init скрипт — вызывать Python вместо bash
- [ ] Удалить старые shell скрипты после тестирования

**Экономия:** 660+ строк bash кода, упрощение поддержки
