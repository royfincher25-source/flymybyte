# Исправление DNS-спуфинга AI-доменов

## Проблема

DNS-запросы к AI-доменам (chatgpt.com, aistudio.google.com и др.) не перенаправляются через VPN, а используют внешний DNS (Cloudflare 1.1.1.1), что не позволяет обойти региональные блокировки.

### Симптомы

```powershell
PS C:\Users\User> nslookup chatgpt.com 192.168.1.1
Сервер:  UnKnown
Address:  192.168.1.1

Не заслуживающий доверия ответ:
Имя:    chatgpt.com
Addresses:  2a06:98c1:3100::6812:202f
          2a06:98c1:310b::ac40:9bd1
          8.6.112.6
          8.47.69.6
```

IP-адреса `8.6.112.6` и `8.47.69.6` принадлежат Cloudflare, а не вашему VPN.

## Причина

В конфигурации DNS-спуфинга был захардкожен **Cloudflare DNS (1.1.1.1:53)** вместо **VPN DNS (127.0.0.1:40500)**.

## Решение

### Вариант 1: Через веб-интерфейс (рекомендуется)

1. Откройте веб-интерфейс: `http://192.168.1.1:8080`
2. Перейдите в **Сервис → DNS-обход AI**
3. Нажмите **"Загрузить готовый список"** → **"AI-сервисы"**
4. Нажмите **"Применить конфигурацию"**
5. Дождитесь перезапуска dnsmasq

### Вариант 2: Вручную через SSH

1. Подключитесь к роутеру по SSH:
   ```bash
   ssh admin@192.168.1.1
   ```

2. Проверьте текущую конфигурацию:
   ```bash
   cat /opt/etc/unblock-ai.dnsmasq
   ```
   
   Если видите `server=/domain.com/1.1.1.1` — конфигурация устарела.

3. Запустите скрипт диагностики:
   ```bash
   sh /opt/root/check_dns_spoofing.sh
   ```

4. Обновите конфигурацию через веб-интерфейс (см. Вариант 1)

5. Перезапустите dnsmasq:
   ```bash
   /opt/etc/init.d/S56dnsmasq restart
   ```

### Вариант 3: Полная переустановка

Если обновление не помогло:

1. В веб-интерфейсе перейдите в **Установка**
2. Нажмите **"Переустановить"**
3. Дождитесь завершения установки
4. Перейдите в **DNS-обход AI** и примените конфигурацию

## Технические детали

### Архитектура DNS-спуфинга

```
Пользователь → dnsmasq (192.168.1.1:53) 
                    ↓
            unblock-ai.dnsmasq
            server=/chatgpt.com/127.0.0.1#40500
                    ↓
         DNS-over-TLS (127.0.0.1:40500)
                    ↓
              VPN туннель
                    ↓
           Внешний DNS (через VPN)
                    ↓
           IP-адреса из другой страны
```

### Файлы конфигурации

| Файл | Назначение |
|------|------------|
| `/opt/etc/unblock-ai.dnsmasq` | DNS правила для AI-доменов |
| `/opt/etc/unblock/ai-domains.txt` | Список AI-доменов |
| `/opt/etc/dnsmasq.conf` | Основная конфигурация dnsmasq |
| `/opt/etc/web_ui/.env` | Настройки веб-интерфейса (порт DNS) |

## Часто задаваемые вопросы

### Q: Почему IPv6 адреса в ответе?

A: dnsmasq возвращает как IPv4, так и IPv6 адреса. Если ваш провайдер не поддерживает IPv6, добавьте в `/opt/etc/dnsmasq.conf`:

```conf
filter-aaaa
filter-aaaa-on-local
```

### Q: Какой DNS порт используется?

A: По умолчанию `40500` (DNS-over-TLS). Можно изменить в веб-интерфейсе при установке.

### Q: Сколько доменов в списке?

A: В стандартном списке ~150 AI-доменов (Google AI, OpenAI, Anthropic, Microsoft AI и др.)

### Q: Можно ли добавить свои домены?

A: Да, отредактируйте `/opt/etc/unblock/ai-domains.txt` и нажмите "Применить" в веб-интерфейсе.
