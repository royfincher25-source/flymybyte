## Анализ best practices (GitHub, Reddit, OpenWrt)

### Популярные подходы

| Проект | Метод | Плюсы | Минусы |
|--------|-------|-------|--------|
| **AdGuard Home** | DNS-based + proactive cache | Автоматически, надежно | Больше RAM |
| **Pi-hole** | DNS only, no ipset | Просто | Только DNS |
| **OpenWrt bypass-scripts** | Hybrid (DNS + static CIDR) | Баланс | Сложность |
| **sing-box** | Inline routing | Быстро | Только для sing-box |

### Рекомендуемые паттерны

1. **Proactive DNS resolution** — резолвить домены при добавлении в список
2. **Single mechanism** — один способ добавления (не два параллельных)
3. **TTL-based caching** — кэшировать IP с учётом TTL
4. **Fallback** — если DNS не работает, использовать последние известные IP

## Рекомендация

**Вариант A** — оптимальный баланс простоты и функциональности:

1. Убрать `ipset=` директивы из unblock.dnsmasq
2. Все домены резолвить при добавлении
3. Добавить лимит (напр. 5000 IP на ipset)
4. Периодический refresh по cron (каждые 30 мин)
5. Сохранить fallback — если resolve не работает, оставить для dnsmasq
