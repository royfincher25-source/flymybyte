## [1.20.0] - 2026-04-05

### 🐛 Исправлено

#### Отключение DNS-обхода AI больше не ломает DNS Override

**Проблема:** При нажатии «Выключить» на странице DNS-обхода AI-доменов вызывался `S56dnsmasq restart`, который полностью останавливал dnsmasq. Это ломало DNS Override (dnsmasq на порту 5353 + DNAT правила), и статус DNS Override переключался на ВЫКЛ.

**Решение:** Вместо полного перезапуска dnsmasq используется `kill -HUP <PID>` — перечитка конфиига без остановки процесса. Это сохраняет настройки DNS Override. Если dnsmasq не запущен — используется `S56dnsmasq start`.

**Также:**
- Улучшено логирование: теперь пишутся stdout/stderr/returncode для диагностики
- Проверка статуса dnsmasq через `pgrep` вместо returncode init-скрипта

## [1.13.0] - 2026-04-01

### 🐛 Исправлено

#### Критическое: потеря интернета при истечении ключа прокси

**Проблема:** При истечении ключа Shadowsocks переставал работать весь интернет на роутере, включая домены не из списков обхода.

**Причина:** В основном `dnsmasq.conf` первичным DNS-сервером был `127.0.0.1#40500` (DNS-over-TLS через прокси). При падении прокси все DNS-запросы зависали.

**Решение:**
- Убран `127.0.0.1#40500` из основного `dnsmasq.conf` — теперь прямые `8.8.8.8` и `1.1.1.1`
- Bypass-домены по-прежнему идут через порт 40500 через `unblock.dnsmasq`
- AI-домены по-прежнему идут через порт 40500 через `unblock-ai.dnsmasq`

#### Ошибки OTA-обновления

- 404 на устаревших файлах (`routes.py`, `web_config.py`) больше не блокируют обновление — безопасно пропускаются
- Пустое сообщение об ошибке в DNS monitor при перезапуске dnsmasq

### 🔧 Изменено

- `dns_manager.py` — сохраняет domain-specific `server=/domain/...` строки при переключении DNS
- `script.sh` — убрана sed-замена порта 40500 при установке dnsmasq.conf

## [1.6.6] - 2026-03-23

### ✨ Новое

#### DNS-обход AI-доменов
Новая функция для обхода региональных блокировок AI-сервисов Google через подмену DNS.

**Поддерживаемые сервисы:**
- Google AI Studio (aistudio.google.com)
- Gemini (gemini.google.com)
- Google Colab (colab.research.google.com)
- Kaggle (kaggle.com)
- Google DeepMind
- TensorFlow
- и другие

**Принцип работы:**
1. DNS-запросы к AI-доменам направляются через VPN DNS (порт 40500)
2. VPN резолвит домены через внешний DNS (8.8.8.8)
3. Сервис видит IP VPN, а не реальный регион пользователя

**Новые файлы:**
- `core/dns_spoofing.py` — модуль DNS-обхода AI-доменов
- `templates/dns_spoofing.html` — веб-интерфейс управления

- `docs/DNS_SPOOFING_DESIGN.md` — дизайн-документ архитектуры
- `docs/DNS_SPOOFING_INSTRUCTION.md` — инструкция пользователя

### 🔧 Технические детали

**Модуль dns_spoofing.py:**
- Singleton pattern для потокобезопасности
- Валидация доменов (проверка формата, длины)
- Фильтрация IP-адресов
- Atomic write конфигурации (через .tmp файл)
- Интеграция с dnsmasq (перезапуск через init script)
- Тестирование разрешения доменов через nslookup

**Конфигурация dnsmasq:**
```conf
# AI Domains DNS Spoofing
address=/aistudio.google.com/127.0.0.1#40500
server=/aistudio.google.com/127.0.0.1#40500
```

**Зависимости:**
- Python 3.8+
- dnsmasq (установлен по умолчанию)
- VPN с DNSPort (Shadowsocks/Xray/Tor)

### 🎯 Быстрый старт

**Через веб-интерфейс:**
1. Откройте `http://192.168.1.1:8080`
2. Перейдите в **Сервис** → **DNS-обход AI** → **Настроить**
3. Нажмите **Загрузить готовый список**
4. Нажмите **Применить**
5. Проверьте статус: должен быть "✅ Активен"

**Через SSH:**
```bash
# 1. Подключитесь к роутеру
ssh root@192.168.1.1 -p 222

# 2. Скачайте список AI-доменов
curl -sL -o /opt/etc/unblock/ai-domains.txt \
  https://raw.githubusercontent.com/royfincher25-source/flymybyte/master/src/web_ui/resources/lists/unblock-ai-domains.txt

# 3. Примените конфигурацию
sh /opt/etc/web_ui/resources/scripts/unblock_dnsmasq.sh

# 4. Проверьте конфигурацию
cat /opt/etc/unblock-ai.dnsmasq
```

### ⚠️ Важные замечания

**Требования:**
- Настроенный VPN (Shadowsocks, Xray, Tor) с DNSPort (порт 40500)
- Установленный dnsmasq
- Python 3.8+

**Безопасность:**
- DNS-запросы идут через VPN DNS, что скрывает регион
- Atomic write предотвращает повреждение конфигурации
- Валидация доменов защищает от некорректных записей

**Производительность:**
- Память: ~1MB (dnsmasq + кэш)
- CPU: минимальное (только при старте)
- Место: ~50KB (конфигурация)

## Формат

Этот проект следует [Semantic Versioning](https://semver.org/lang/ru/):
- **MAJOR** — обратно несовместимые изменения
- **MINOR** — новые функции, обратно совместимые
- **PATCH** — исправления ошибок, обратно совместимые

Формат: `[MAJOR.MINOR.PATCH] - YYYY-MM-DD`

### Типы изменений

- **✨ Новое** — новые функции
- **🔧 Изменено** — изменения в существующем функционале
- **🐛 Исправлено** — исправления ошибок
