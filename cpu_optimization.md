# Диагностика и решение проблемы высокой нагрузки на CPU

## Проблема
После обновления FlyMyByte на роутере Keenetic сильно нагружается процессор.

## Причина
Основная причина — **DNSMonitor** (фоновый поток мониторинга DNS), который:
1. Проверял DNS-сервер **каждые 60 секунд**
2. При проверке использовал **таймаут 3 секунды** на каждый сервер
3. При смене DNS выполнял множество проверок подряд
4. Постоянное логирование каждого check создавало дополнительную I/O нагрузку

## Решение (версия 2.5.9)

### 1. Увеличен интервал проверки DNS
- **Было:** `DNS_CHECK_INTERVAL = 60` секунд
- **Стало:** `DNS_CHECK_INTERVAL = 300` секунд (5 минут)
- **Эффект:** CPU нагрузка от DNS мониторинга снижена на **~80%**

### 2. Оптимизирован таймаут DNS
- **Было:** `DNS_TIMEOUT = 3` секунды
- **Стало:** `DNS_TIMEOUT = 2` секунды
- **Эффект:** Более быстрое определение недоступности DNS

### 3. Увеличен порог переключения DNS
- **Было:** `DNS_FAILURE_THRESHOLD = 3`
- **Стало:** `DNS_FAILURE_THRESHOLD = 5`
- **Эффект:** Меньше ложных переключений и лишних проверок

### 4. Оптимизировано логирование
- Успешные DNS проверки логируются только **каждый 10-й раз**
- Снижена нагрузка на I/O подсистему роутера

### 5. Улучшено управление сокетами
- Явное закрытие сокетов в блоках `except`
- Предотвращение утечек ресурсов

## Диагностика на роутере

Для проверки нагрузки выполните на роутере:

```bash
sh H:\disk_e\dell\FlyMyByte\test.txt
```

Или скопируйте скрипт на роутер и выполните:
```bash
scp test.txt root@192.168.1.1:/opt/tmp/
ssh root@192.168.1.1 "sh /opt/tmp/test.txt"
```

## Что проверить после обновления

1. **Нагрузка CPU:**
   ```bash
   top -bn1 | head -15
   ```
   Python процесс должен использовать **< 5%** CPU в среднем

2. **DNS Monitor статус:**
   - Откройте веб-интерфейс → Сервисы → DNS Monitor
   - Должен показывать текущий DNS сервер и последнее время проверки

3. **Логи:**
   ```bash
   tail -50 /opt/var/log/web_ui.log
   ```
   DNS проверки должны появляться раз в 5 минут

## Дополнительные рекомендации

### Если нагрузка всё ещё высокая:

1. **Проверьте количество сервисов:**
   ```bash
   ps | grep -E "(xray|hysteria|shadowsocks|trojan|tor)" | grep -v grep
   ```

2. **Проверьте ipset размеры:**
   ```bash
   ipset list -n | while read name; do
       count=$(ipset list "$name" 2>/dev/null | grep -c "^[0-9]")
       echo "  $name: $count entries"
   done
   ```

3. **Проверьте dnsmasq:**
   ```bash
   ps | grep dnsmasq | grep -v grep
   logread | grep dnsmasq | tail -20
   ```

## Технические детали

### Архитектура DNSMonitor

```
DNSMonitor (singleton)
  ├─ _monitor_loop() — фоновый поток
  │   ├─ check_dns_server() — проверка текущего DNS (каждые 5 мин)
  │   ├─ _select_best_primary() — выбор лучшего DNS (при старте/сбое)
  │   └─ _switch_to_backup() — переключение на backup (при сбое)
  └─ get_status() — статус для веб-интерфейса
```
